# 盈利预测功能 — 设计规格

## 概述

在现有 5 个分析区块的基础上，新增「盈利预测」能力：对每只筛选/持仓基金，基于技术指标特征向量进行历史模式匹配，统计 1 月/2 月/3 月盈利概率和幅度，再由 DeepSeek AI 结合市场情绪做修正解读。

## 核心架构

```
365天历史数据 → 6维特征提取 → 滚动窗口特征库 → 余弦相似度匹配(Top30)
                                         ↓
沪深300基准 → 相对强弱     → 统计分布(概率/中位数/置信区间)
                                         ↓
                               DeepSeek AI 修正 → 三层UI展示
```

## 数据层

### 新增数据获取 ([data/fund_fetcher.py](data/fund_fetcher.py))

- `fetch_benchmark_history(days=365)` — 获取沪深300 ETF（510300）的 NAV 历史作为基准，复用现有 `fetch_fund_nav_history("510300", days)`。

### 新增技术指标 ([data/technical.py](data/technical.py))

| 函数 | 说明 |
|------|------|
| `calc_rsi(df, period=14)` | 相对强弱指标，返回 0-100 Series |
| `calc_bollinger(df, period=20, std=2)` | 布林带 upper/middle/lower 三轨 |
| `calc_volatility(df, period=20)` | 年化波动率（标准差 × √252） |
| `extract_feature_vector(df, benchmark_df)` | 6维特征向量，归一化到 [0,1] |

**6 维特征向量定义：**

| 维度 | 指标 | 权重含义 |
|------|------|----------|
| F1 | MA 排列得分 | (NAV-MA5)/MA5, (MA5-MA10)/MA10, (MA10-MA20)/MA20 的加权均值 |
| F2 | MACD 动量 | DIF - DEA 的正负与幅度，经 sigmoid 压缩 |
| F3 | RSI 位置 | (RSI-30)/40，裁剪到 [0,1] |
| F4 | 布林带位置 | (价格-下轨)/(上轨-下轨)，裁剪到 [0,1] |
| F5 | 年化波动率 | 20日波动率，min/max 归一化 |
| F6 | 基准相对强弱 | 基金 20 日收益 - 基准 20 日收益，tanh 压缩 |

## 分析层

### 预测引擎 ([analysis/predictor.py](analysis/predictor.py)) — 新文件

**核心函数：**

```
predict_fund_return(code, days=365) -> PredictionResult
```

**处理流程：**

1. 获取基金 365 天 NAV 历史 + 沪深300 同期数据
2. 计算当前交易日的 6 维特征向量 V_now
3. 滚动窗口：从第 60 天起，每个交易日计算一个快照 (特征向量 V_t, 后续_22d_return, 后续_44d_return,_后续_66d_return)
4. 用余弦相似度找出与 V_now 最相似的 30 个历史快照
5. 对 30 个样本的后续收益做统计：
   - 盈利概率 = 正收益样本数 / 30
   - 预期收益中位数 = median(30个收益)
   - 置信区间 = [25th percentile, 75th percentile]
   - 最大收益 / 最大亏损 = 极值参考
6. 返回 `PredictionResult` dataclass

**数据类：**

```python
@dataclass
class PeriodPrediction:
    win_probability: float      # 盈利概率 0-1
    median_return: float        # 预期收益中位数
    p25_return: float           # 25% 分位
    p75_return: float           # 75% 分位
    max_gain: float             # 历史相似情景最大收益
    max_loss: float             # 历史相似情景最大亏损

@dataclass
class PredictionResult:
    code: str
    name: str
    current_features: dict      # 当前特征值
    match_count: int            # 有效匹配数
    avg_similarity: float       # 平均相似度
    pred_1m: PeriodPrediction
    pred_2m: PeriodPrediction
    pred_3m: PeriodPrediction
    confidence: str             # "高"/"中"/"低" — 基于相似度判定
```

### AI 预测修正 ([analysis/recommender.py](analysis/recommender.py))

## 性能考量

- 预测计算耗时（每只基金需要 365 天数据 + 约 300 个滚动窗口的特征提取 + 相似度计算）约 2-3 秒
- 10 只基金需约 20-30 秒
- 使用缓存：365 天 NAV + benchmark 数据缓存 4 小时；单只基金的滚动特征库缓存 2 小时
- 在筛选区默认关闭预测，勾选后在后台 spinner 中计算
- 持仓诊断区（通常 2-5 只基金）自动启用预测

## 边界条件处理

- **历史数据不足**（< 60 个交易日）：返回 `PredictionResult` 但所有 `PeriodPrediction` 设为 `None`，UI 显示「数据不足，无法预测」
- **匹配相似度极低**（< 0.5）：置信度标记为「低」，UI 显示警告提示
- **基金成立不足 1 年**：使用全部可用数据，窗口至少需要 60 个交易日
- **单边极端行情**（30 个相似样本全部同向）：flag 标注「趋势一致性高」
